# Doc Site Using Docusaurus

🔗 **Live Site:** https://lizmarlowe-byte.github.io/docs-as-code-portfolio-docusaurus/

Welcome to the second project in my **Docs‑as‑Code Portfolio Series**, where I’m building and publishing documentation sites using four different static site generators (SSGs): **MkDocs**, **Hugo**, **Docusaurus**, and **Jekyll**.

This repo represents my work with **Docusaurus** and demonstrates how I approach modern documentation workflows—from information architecture and content strategy to build automation and publishing.

## 🎯 Purpose of This Project

As a Sr. Technical Writer, I use git‑based workflows and developer‑centric tooling.

This portfolio series is my opportunity to:

* Strengthen and demonstrate my docs‑as‑code skills
* Explore and compare popular SSGs used in engineering teams
* Showcase real examples of my technical writing
* Build documentation sites end‑to‑end using best practices
* Create a curated, public‑facing writing and tooling portfolio


## ⚙️ What I Built Using **Docusaurus**

### 🔧 Workflow & Architecture
- Content authored in Markdown/MDX
- Information architecture organized using autogenerated sidebars via `_category_.json`  
- Clean hierarchical structure for guides, APIs, how‑tos, and reference docs  
- Light‑only theme to match my portfolio aesthetic  
- Consistent URL structure using explicit `slug` definitions for category landing pages  

### 🚀 Build & Delivery
- Local development with `npm start`  
- Production builds with `npm run build`  
- Deployable to GitHub Pages with Docusaurus’s built‑in `docusaurus deploy`  
- Link safety enforced through Docusaurus’s broken‑link validation  
- Category landing pages generated automatically via `"type": "generated-index"`
